定义于头文件 <experimental/scope>
template<class R, class D>
class unique_resource;
 (库基础 TS v3)

unique_resource 是通用的资源柄 RAII 包装，通过柄占有并管理资源，并在 unique_resource 销毁时释放该资源。

在以下任一情况发生时，用 D 类型删除器释放资源：

  • 销毁管理的 unique_resource 对象
  • 经由 operator= 或 reset() 赋另一资源给 unique_resource 对象。

若 R 为对象类型则令 RS 为 R ，否则为 std::reference_wrapper<std::remove_reference_t<R>> ：

  • unique_resource 等效地保有包装或作为资源柄的 RS 类型子对象、 D 类型删除器及指示包装是否占有资源的 bool 标志。
  • 为解释目的，称 RS 为存储的资源柄，存储（若 R 为对象类型）或包装（若 R 为引用类型）的 R 为底层资源柄。 LFTS 不使用此二用语。

模板形参

R - 资源柄类型
D - 删除器类型
类型要求
-
R 应当为对象类型或到对象的左值引用类型。令 UnrefR 为 std::remove_reference_t<R> ，则 UnrefR 应当为可移动构造 (MoveConstructible) ，而若 UnrefR 非可复制构造 (CopyConstructible) ，则 std::is_nothrow_move_constructible_v<UnrefR> 应当为 true 。
-
D 应当为可析构 (Destructible) 且可移动构造 (MoveConstructible) 的函数对象 (FunctionObject) 类型，而若 D 非可复制构造 (CopyConstructible) ，则 std::is_nothrow_move_constructible_v<R> 应当为 true 。 给定 D 类型左值 d 与 UnrefR 类型左值 r ，表达式 d(r) 应当为良构。

注解

满足可空指针 (NullablePointer) 的资源柄类型亦能由 std::unique_ptr 管理。不同于 unique_ptr ， unique_resource 不要求可空指针 (NullablePointer) 。
